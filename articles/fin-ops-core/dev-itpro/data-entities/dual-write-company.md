## <a name="company-concept-in-common-data-service"></a><span data-ttu-id="d834a-103">Concetto di società in Common Data Service</span><span class="sxs-lookup"><span data-stu-id="d834a-103">Company concept in Common Data Service</span></span>

[!include [banner](../includes/banner.md)]

<span data-ttu-id="d834a-104">In Finance and Operations, il concetto di *società* è un costrutto sia legale che commerciale.</span><span class="sxs-lookup"><span data-stu-id="d834a-104">In Finance and Operations, the concept of a *company* is both a legal construct and a business construct.</span></span> <span data-ttu-id="d834a-105">È inoltre un limite di visibilità e di sicurezza per i dati.</span><span class="sxs-lookup"><span data-stu-id="d834a-105">It's also a security and visibility boundary for data.</span></span> <span data-ttu-id="d834a-106">Gli utenti lavorano sempre nel contesto di una singola società e la maggior parte dei dati è oggetto dello striping della società.</span><span class="sxs-lookup"><span data-stu-id="d834a-106">Users always work in the context of a single company, and most of the data is striped by company.</span></span>

<span data-ttu-id="d834a-107">Common Data Service non dispone di un concetto equivalente.</span><span class="sxs-lookup"><span data-stu-id="d834a-107">Common Data Service doesn't have an equivalent concept.</span></span> <span data-ttu-id="d834a-108">Il concetto più vicino è *Business Unit*, ovvero principalmente un limite di visibilità e sicurezza per i dati utente.</span><span class="sxs-lookup"><span data-stu-id="d834a-108">The closest concept is *business unit*, which is primarily a security and visibility boundary for user data.</span></span> <span data-ttu-id="d834a-109">Questo concetto non ha le stesse implicazioni legali o commerciali del concetto di società.</span><span class="sxs-lookup"><span data-stu-id="d834a-109">This concept doesn't have the same legal or business implications as the company concept.</span></span>

<span data-ttu-id="d834a-110">Poiché Business Unit e società non sono equivalenti concetti, non è possibile applicare un mapping di uno-a-uno (1:1) tra loro a scopo dell'integrazione con Common Data Service.</span><span class="sxs-lookup"><span data-stu-id="d834a-110">Because business unit and company aren't equivalent concepts, it isn't possible to force a one-to-one (1:1) mapping between them for the purpose of Common Data Service integration.</span></span> <span data-ttu-id="d834a-111">Tuttavia, poiché gli utenti devono, per impostazione predefinita, poter visualizzare gli stessi record nell'applicazione e in Common Data Service, Microsoft ha introdotto una nuova entità in Common Data Service denominata cdm\_Company.</span><span class="sxs-lookup"><span data-stu-id="d834a-111">However, because users must, by default, be able to see the same records in the application and Common Data Service, Microsoft has introduced a new entity in Common Data Service that is named cdm\_Company.</span></span> <span data-ttu-id="d834a-112">Questa entità equivale all'entità Società nell'applicazione.</span><span class="sxs-lookup"><span data-stu-id="d834a-112">This entity is equivalent to the Company entity in the application.</span></span> <span data-ttu-id="d834a-113">Per contribuire a garantire che la visibilità dei record è equivalente tra l'applicazione e Common Data Service in modo predefinito, è consigliabile la seguente impostazione per i dati in Common Data Service:</span><span class="sxs-lookup"><span data-stu-id="d834a-113">To help guarantee that visibility of records is equivalent between the application and Common Data Service out of the box, we recommend the following setup for data in Common Data Service:</span></span>
