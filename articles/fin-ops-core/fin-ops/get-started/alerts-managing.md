### <a name="frequency-for-change-based-events"></a><span data-ttu-id="64efb-116">Frequenza per gli eventi basati sulla modifica</span><span class="sxs-lookup"><span data-stu-id="64efb-116">Frequency for change-based events</span></span>

<span data-ttu-id="64efb-120">Un processo batch che si ripete invece con minore frequenza ed è pianificato per orari in cui il carico del sistema è ridotto può contribuire a migliorare le prestazioni del sistema.</span><span class="sxs-lookup"><span data-stu-id="64efb-120">On the other hand, a batch job that recurs less often, and that is scheduled for times when the system load is low, might help improve system performance.</span></span> <span data-ttu-id="64efb-121">Tuttavia, una frequenza bassa dell'elaborazione batch può non soddisfare le richieste degli utenti riguardanti la tempestività degli avvisi.</span><span class="sxs-lookup"><span data-stu-id="64efb-121">However, a low frequency for batch processing might not meet the users' demands for timely alerts.</span></span>

<span data-ttu-id="64efb-128">Se si imposta una frequenza bassa per l'elaborazione batch degli eventi basati sulla modifica, i dati rilevanti per le condizioni delle regole di avviso possono cambiare prima dell'elaborazione del batch.</span><span class="sxs-lookup"><span data-stu-id="64efb-128">If you set up a low frequency for batch processing for change-based events, data that is relevant to the conditions in alert rules might be changed before the batch is processed.</span></span> <span data-ttu-id="64efb-129">Di conseguenza, è possibile perdere degli avvisi.</span><span class="sxs-lookup"><span data-stu-id="64efb-129">Therefore, you might lose alerts.</span></span>

<span data-ttu-id="64efb-130">Ad esempio una regola di avviso viene impostata per generare un avviso quando si verifica l'evento **modifica del contatto del cliente** e viene soddisfatta la condizione **cliente = BB**.</span><span class="sxs-lookup"><span data-stu-id="64efb-130">For example, an alert rule is set up to trigger an alert when the event is **customer contact changes** and the condition is **customer = BB**.</span></span> <span data-ttu-id="64efb-131">In altre parole, quando il contatto del cliente BB cambia, l'evento viene registrato.</span><span class="sxs-lookup"><span data-stu-id="64efb-131">In other words, when the customer contact for customer BB is changed, the event is logged.</span></span> <span data-ttu-id="64efb-132">Tuttavia, il sistema di elaborazione batch viene impostato in modo che l'elaborazione batch si verifichi con una frequenza minore rispetto all'immissione dei dati.</span><span class="sxs-lookup"><span data-stu-id="64efb-132">However, the batch processing system is set up so that batch processing occurs less often than data entry.</span></span> <span data-ttu-id="64efb-133">Se il nome del cliente viene modificato da **BB** in **AA** prima che l'evento venga elaborato, i dati nel database non corrispondono più alla condizione della regola, **cliente = BB**.</span><span class="sxs-lookup"><span data-stu-id="64efb-133">If the customer name is changed from **BB** to **AA** before the event is processed, the data in the database no longer matches the condition in the rule, **customer = BB**.</span></span> <span data-ttu-id="64efb-134">Di conseguenza, quando l'evento è infine elaborato, non viene generato alcun avviso.</span><span class="sxs-lookup"><span data-stu-id="64efb-134">Therefore, when the event is finally processed, no alert is generated.</span></span>
