[!include [banner](../includes/banner.md)]

<span data-ttu-id="6edd2-104">In questo argomento vengono fornite informazioni sui parametri e sui filtri utilizzati nel calcolo di previsione della domanda.</span><span class="sxs-lookup"><span data-stu-id="6edd2-104">This topic provides information about the parameters and filters that are used in the calculation of demand forecasting.</span></span>

<span data-ttu-id="6edd2-107">Per generare una previsione di base statistica, andare a **Pianificazione generale &gt;  Previsioni &gt; Previsione della domanda &gt; Genera previsione di base statistica**.</span><span class="sxs-lookup"><span data-stu-id="6edd2-107">To generate a demand forecast, go to **Master planning &gt; Forecasting &gt; Demand forecasting &gt; Generate statistical baseline forecast**.</span></span> 

<span data-ttu-id="6edd2-108">L'intervallo di previsione può essere selezionato al momento della generazione della previsione.</span><span class="sxs-lookup"><span data-stu-id="6edd2-108">The forecast bucket can be selected at forecast generation time.</span></span> <span data-ttu-id="6edd2-109">I valori disponibili sono: Giorno, Settimana e Mese.</span><span class="sxs-lookup"><span data-stu-id="6edd2-109">The available values are: Day, Week, and Month.</span></span> 

<span data-ttu-id="6edd2-110">Il numero di intervalli per la generazione della previsione viene impostato nel campo **Orizzonte previsione**.</span><span class="sxs-lookup"><span data-stu-id="6edd2-110">The number of buckets to generate a forecast for is set in the **Forecast horizon** field.</span></span>

-   <span data-ttu-id="6edd2-114">Copiando la domanda dello stesso trimestre dell'anno precedente.</span><span class="sxs-lookup"><span data-stu-id="6edd2-114">By copying the demand from the same quarter last year.</span></span>
-   <span data-ttu-id="6edd2-115">Copiando la domanda del trimestre precedente.</span><span class="sxs-lookup"><span data-stu-id="6edd2-115">By copying the demand from the previous quarter.</span></span>

<span data-ttu-id="6edd2-116">Per evitare confusione nei piani di produzione, è possibile bloccare un certo numero di intervalli di previsione.</span><span class="sxs-lookup"><span data-stu-id="6edd2-116">To prevent confusion in the production plans, a certain number of forecast buckets can be frozen.</span></span> <span data-ttu-id="6edd2-117">Questo numero viene impostato nel campo **Intervallo temporale blocco**.</span><span class="sxs-lookup"><span data-stu-id="6edd2-117">This number is set in the **Freeze time fence** field.</span></span> <span data-ttu-id="6edd2-118">Nella pagina **Previsione della domanda modificata** le celle corrispondenti agli intervalli bloccati vengono disabilitate, per fornire un'indicazione visiva che i valori non devono essere modificati.</span><span class="sxs-lookup"><span data-stu-id="6edd2-118">On the **Adjusted demand forecast** page, the cells for the frozen buckets are disabled, to give a visual indication that those values should not be changed.</span></span> 

<span data-ttu-id="6edd2-119">La data di inizio per la previsione della domanda di base non deve corrispondere alla data corrente o a una data in futuro.</span><span class="sxs-lookup"><span data-stu-id="6edd2-119">The start date for the baseline demand forecast doesn’t have to be the current date or a date in the future.</span></span> <span data-ttu-id="6edd2-120">Per impostare una data di inizio diversa, utilizzare il campo **Data di inizio previsione di base - Da**.</span><span class="sxs-lookup"><span data-stu-id="6edd2-120">To set a different start date, use the **Baseline forecast start date - From date** field.</span></span> <span data-ttu-id="6edd2-121">Ad esempio, a giugno, gli utenti possono generare una previsione per l'anno successivo.</span><span class="sxs-lookup"><span data-stu-id="6edd2-121">For example, in June, users can generate a forecast for the next year.</span></span> <span data-ttu-id="6edd2-122">Poiché gli intervalli di previsione tra la fine della richiesta storica e l'inizio di base mancano, le previsioni potrebbero non essere precise.</span><span class="sxs-lookup"><span data-stu-id="6edd2-122">Because the forecast buckets between the end of historical demand and the start of the baseline are missing, the predictions might not be accurate.</span></span> <span data-ttu-id="6edd2-123">Se si utilizza il servizio di previsione della domanda di Microsoft Dynamics 365 for Finance and Operations, sono disponibili quattro modalità in cui è possibile compilare le interruzioni mancanti.</span><span class="sxs-lookup"><span data-stu-id="6edd2-123">If you are using the Microsoft Dynamics 365 for Finance and Operations Demand forecasting service, there are four ways in which you can fill in the missing gaps.</span></span> <span data-ttu-id="6edd2-124">È possibile selezionare il metodo di registrazione desiderato impostando il parametro MISSING\_VALUE\_SUBSTITUTION nella pagina **Parametri di previsione della domanda**.</span><span class="sxs-lookup"><span data-stu-id="6edd2-124">You can choose the method that you want by setting the MISSING\_VALUE\_SUBSTITUTION parameter on the **Demand forecasting parameters** page.</span></span> 

> [!NOTE]
> <span data-ttu-id="6edd2-125">La sostituzione dei valori mancanti viene eseguita solo per le interruzioni nei dati tra le date di inizio e fine dei dati storici.</span><span class="sxs-lookup"><span data-stu-id="6edd2-125">Missing value substitution only works for the gaps in data in between the start and end dates for historical data.</span></span> <span data-ttu-id="6edd2-126">Non specificherà i dati prima o dopo l'ultimo punto dati fisico, in quanto funge solo da estrapolazione tra i punti dati esistenti effettivi.</span><span class="sxs-lookup"><span data-stu-id="6edd2-126">It will not fill in data before or after the last physical data point, it only acts as extrapolation between actual existing data points.</span></span> 

<span data-ttu-id="6edd2-127">Il campo **Data di inizio previsione di base** - **Dal** deve essere impostato all'inizio di un intervallo di previsione, ad esempio negli Stati Uniti questo giorno corrisponderà alla domenica se l'intervallo di previsione è settimanale.</span><span class="sxs-lookup"><span data-stu-id="6edd2-127">The **Baseline forecast start date** - **From date** field has to be set to the beginning of a forecast bucket, for example, in the United States, a Sunday if the forecasting bucket is the week.</span></span> <span data-ttu-id="6edd2-128">Il sistema regola automaticamente il campo **Data di inizio previsione di base**  - **Dal** in modo che corrisponda all'inizio di un intervallo di previsione.</span><span class="sxs-lookup"><span data-stu-id="6edd2-128">The system automatically adjusts the **Baseline forecast start date** - **From date** field to match the beginning of a forecast bucket.</span></span> 

<span data-ttu-id="6edd2-129">Il campo **Data di inizio previsione di base** - **Dal** può essere impostato su una data nel passato.</span><span class="sxs-lookup"><span data-stu-id="6edd2-129">The **Baseline forecast start date** - **From date** field can be set to a date in in the past.</span></span> <span data-ttu-id="6edd2-130">Ovvero è possibile generare una previsione della domanda nel passato.</span><span class="sxs-lookup"><span data-stu-id="6edd2-130">In other words, it is possible to generate a demand forecast in the past.</span></span> <span data-ttu-id="6edd2-131">Questa opzione è utile, poiché consente agli utenti di modificare i parametri del servizio di previsione in modo che la previsione statistica generata nel passato corrisponda alla richiesta storica effettiva.</span><span class="sxs-lookup"><span data-stu-id="6edd2-131">This is useful, because it lets users adjust the forecast service parameters so that the statistical forecast generated in the past matches the actual historical demand.</span></span> <span data-ttu-id="6edd2-132">Gli utenti possono quindi continuare a utilizzare le impostazioni dei parametri per generare una previsione di base statistica per il futuro.</span><span class="sxs-lookup"><span data-stu-id="6edd2-132">Users can then continue using these parameter settings to generate a statistical baseline forecast for the future.</span></span> 

<span data-ttu-id="6edd2-133">Le rettifiche manuali effettuate nelle iterazioni precedenti di previsione della domanda possono essere applicati automaticamente alla nuova previsione di base se è data-ttu-id="6edd2-134">Se la casella di controllo è deselezionata, le correzioni manuali non vengono aggiunte alla previsione di base, ma non verranno eliminate.</span><span class="sxs-lookup"><span data-stu-id="6edd2-134">If the check box is cleared, the manual adjustments are not added to the baseline forecast – but they are not deleted.</span></span> <span data-ttu-id="6edd2-135">Le correzioni **Save the manual adjustments made to the baseline demand forecast** check box.</span></span> <span data-ttu-id="6edd2-136">Le correzioni manuali vengono salvate nella fase di autorizzazione.</span><span class="sxs-lookup"><span data-stu-id="6edd2-136">Manual adjustments are saved at authorization time.</span></span> <span data-ttu-id="6edd2-137">Di conseguenza, se un 

<span data-ttu-id="6edd2-139">Una creazione di una previsione della domanda può avere un nome e dei commenti per facilitare gli utenti a identificare la previsione generata.</span><span class="sxs-lookup"><span data-stu-id="6edd2-139">A demand forecast generation can have a name and comments to help users identify the forecast that has been generated.</span></span> <span data-ttu-id="6edd2-140">Questi valori sono visibili nello storico della generazione di previsione nella pagina **Storico generazione previsione di base statistica**.</span><span class="sxs-lookup"><span data-stu-id="6edd2-140">These values are visible in forecast generation history on the **Statistical baseline forecast generation history** page.</span></span>
