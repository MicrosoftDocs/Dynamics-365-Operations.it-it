# <a name="wave-allocation"></a><span data-ttu-id="9dee3-103">Allocazione ciclo</span><span class="sxs-lookup"><span data-stu-id="9dee3-103">Wave allocation</span></span>

[!include [banner](../includes/banner.md)]

<span data-ttu-id="9dee3-104">L'elaborazione del ciclo può richiedere molto tempo e la maggior parte del tempo viene impiegata nella fase di allocazione e nella fase di creazione del lavoro.</span><span class="sxs-lookup"><span data-stu-id="9dee3-104">Wave processing can be time consuming, and most of the processing time is spent in the allocation step and in the work creation step.</span></span>

<span data-ttu-id="9dee3-105">È ora possibile eseguire ciascuna di queste fasi in parallelo, il che può migliorare le prestazioni dell'elaborazione del ciclo e consentire una maggiore produttività dei cicli nello stesso magazzino.</span><span class="sxs-lookup"><span data-stu-id="9dee3-105">It is now possible to run each of these steps in parallel, which can improve the performance of the wave processing, and allow for a larger throughput of waves in the same warehouse.</span></span> <span data-ttu-id="9dee3-106">Questo argomento spiega come impostare il metodo di allocazione del ciclo per l'esecuzione in parallelo.</span><span class="sxs-lookup"><span data-stu-id="9dee3-106">This topic explains how to set up the wave allocation method to run in parallel.</span></span> <span data-ttu-id="9dee3-107">Per ulteriori informazioni su come impostare la creazione del lavoro per l'esecuzione in parallelo, vedi [Programmare la creazione del lavoro durante il ciclo](configure-wave-schedule-work-creation.md).</span><span class="sxs-lookup"><span data-stu-id="9dee3-107">For more information about how to set up work creation to run in parallel, see [Schedule work creation during wave](configure-wave-schedule-work-creation.md).</span></span>

<span data-ttu-id="9dee3-108">In precedenza era possibile allocare solo un ciclo alla volta in un magazzino.</span><span class="sxs-lookup"><span data-stu-id="9dee3-108">Previously it was only possible to allocate one wave at a warehouse at a time.</span></span> <span data-ttu-id="9dee3-109">Questo vincolo è stato rimosso e sostituito da un nuovo vincolo che blocca solo l'elemento e le dimensioni che si trovano al di sopra dell'ubicazione nella gerarchia di prenotazione.</span><span class="sxs-lookup"><span data-stu-id="9dee3-109">This constraint has been removed and replaced by a new constraint that only locks the item and dimensions that are above location in the reservation hierarchy.</span></span> <span data-ttu-id="9dee3-110">Le dimensioni sopra l'ubicazione includono sempre le dimensioni del prodotto.</span><span class="sxs-lookup"><span data-stu-id="9dee3-110">Dimensions above the location always include product dimensions.</span></span> <span data-ttu-id="9dee3-111">Ad esempio, se un elemento è configurato utilizzando *Colore*, quindi le varianti per *Rosso*, *Blu*, e *Giallo* potrebbero essere elaborate in parallelo.</span><span class="sxs-lookup"><span data-stu-id="9dee3-111">For example, if an item is configured using *Color*, then variants for *Red*, *Blue*, and *Yellow* could each be processed in parallel.</span></span>

<span data-ttu-id="9dee3-112">Ciò significa che se lo stesso oggetto con le stesse dimensioni sopra l'ubicazione viene allocato da un ciclo, gli altri cicli dovranno attendere per acquisire un blocco sullo stesso oggetto e dimensioni.</span><span class="sxs-lookup"><span data-stu-id="9dee3-112">This means that if the same item with the same dimensions above the location is being allocated by one wave, other waves will have to wait to acquire a lock on the same item and dimensions.</span></span> <span data-ttu-id="9dee3-113">Se il blocco non può essere acquisito in modo tempestivo, si verificherà un errore e l'elaborazione del ciclo non riesce.</span><span class="sxs-lookup"><span data-stu-id="9dee3-113">If the lock can't be acquired in a timely manner, an error will occur and the wave processing will fail.</span></span>

## <a name="performance-improvements"></a><span data-ttu-id="9dee3-115">Miglioramenti delle prestazioni</span><span class="sxs-lookup"><span data-stu-id="9dee3-115">Performance improvements</span></span>

- <span data-ttu-id="9dee3-117">**Throughput migliorato** - La velocità effettiva dei cicli in genere migliorerà anche se l'elaborazione parallela non è configurata, soprattutto per gli scenari in cui non vi è sovrapposizione di elementi all'interno dei cicli.</span><span class="sxs-lookup"><span data-stu-id="9dee3-117">**Improved throughput** - The throughput of waves will typically improve even if parallel processing isn't configured, especially for scenarios where there is no overlap of items within the waves.</span></span>
