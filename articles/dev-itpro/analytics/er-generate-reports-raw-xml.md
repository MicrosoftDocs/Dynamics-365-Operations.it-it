[!include[banner](../includes/banner.md)]

<span data-ttu-id="65a4c-104">È possibile utilizzare il nuovo elemento di formato **RAW XML** per progettare formati di report elettronici (ER) che generano i documenti in uscita in formato XML.</span><span class="sxs-lookup"><span data-stu-id="65a4c-104">You can use the new **RAW XML** format element to design Electronic reporting (ER) formats that generate outgoing documents in XML format.</span></span> <span data-ttu-id="65a4c-105">In alcuni casi, è preferibile aggiungere dati XML non elaborati a questi report per uno dei seguenti motivi:</span><span class="sxs-lookup"><span data-stu-id="65a4c-105">In some cases, you might prefer to add raw XML data to these reports for one or more of the following reasons:</span></span>

- <span data-ttu-id="65a4c-106">È più pratico utilizzare il codice XML non elaborato per la progettazione originale e la gestione continua di un report, poiché la struttura XML può essere generata automaticamente eseguendo un'espressione runtime.</span><span class="sxs-lookup"><span data-stu-id="65a4c-106">It's more convenient to use raw XML for the original design and ongoing maintenance of a report, because the XML structure can be automatically generated by executing a runtime expression.</span></span> <span data-ttu-id="65a4c-107">Di conseguenza, non è necessario determinare più associazioni per più elementi di formato in fase di progettazione.</span><span class="sxs-lookup"><span data-stu-id="65a4c-107">Therefore, multiple bindings don't have to be determined for multiple format elements at design time.</span></span> <span data-ttu-id="65a4c-108">È possibile quando le origini dati in uso contengono informazioni che possono essere utilizzate per creare elementi XML quando viene generato il report.</span><span class="sxs-lookup"><span data-stu-id="65a4c-108">It is possible when the data sources that you're using contain information that can be used to make XML elements while the report is generated.</span></span>
- <span data-ttu-id="65a4c-109">Non esiste alcuna altra modalità per compilare il report con contenuto XML precedentemente ricevuto e archiviato nel sistema.</span><span class="sxs-lookup"><span data-stu-id="65a4c-109">No other method can be used to fill the report with XML content that was previously received and stored in the system.</span></span> <span data-ttu-id="65a4c-110">Ad esempio, la risposta XML che viene generata potrebbe dover includere il contenuto di una richiesta XML che è stata inviata in precedenza.</span><span class="sxs-lookup"><span data-stu-id="65a4c-110">For example, the XML response that is generated might have to contain the content of an XML request that was sent earlier.</span></span>
- <span data-ttu-id="65a4c-111">Nessun'altra modalità può essere utilizzata per inserire caratteri nel documento generato in base ai codici numerici.</span><span class="sxs-lookup"><span data-stu-id="65a4c-111">No other method can be used to insert characters into the generated document based on their numeric codes.</span></span> <span data-ttu-id="65a4c-112">Per alcune lingue e alcuni caratteri, i codici di questo tipo non sono disponibili.</span><span class="sxs-lookup"><span data-stu-id="65a4c-112">For some languages and characters, codes of this type don't exist.</span></span> <span data-ttu-id="65a4c-113">Alcuni esempi sono rappresentati dalla lettera greca rho (ρ) e dai codici di entità HTML quali \&eacute; per una *e* che ha l'accento acuto (é).</span><span class="sxs-lookup"><span data-stu-id="65a4c-113">Examples include the Greek letter rho (ρ) and HTML entity codes such as \&eacute; for an *e* that has an acute accent (é).</span></span>

> <span data-ttu-id="65a4c-114">Tenere presente che il framework non verifica se il contenuto XML che viene inserito nel documento generato utilizzando l'elemento di formato **RAW XML** è corretto.</span><span class="sxs-lookup"><span data-stu-id="65a4c-114">Be aware that the framework doesn't control whether the XML content that is placed to the generated document by using the **RAW XML** format element is correct.</span></span>
